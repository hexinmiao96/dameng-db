# 达梦图形化工具完整指南

达梦数据库提供一整套图形化管理工具，覆盖数据库管理、服务管理、数据迁移等运维全场景。以下为各工具的详细使用指南。

---

## DM 管理工具 (manager)

**DM 管理工具** 是达梦数据库的核心图形化管理平台，类似于 Oracle 的 SQL Developer 或 MySQL 的 Workbench。集成了日常管理、SQL 开发、备份恢复、作业调度等主要运维功能。

### 启动方式

| 平台 | 启动方法 |
|------|---------|
| **Windows** | 开始菜单 → 达梦数据库 → DM 管理工具 |
| **Linux** | 进入 DM 安装目录下的 `tool/` 子目录，执行 `./manager` |
| **命令行** | `cd $DM_HOME/tool && ./manager` |

### 功能详解

#### 1. 连接管理

启动后首先需要注册数据库连接：

```
1. 右键" LOCAL" → 注册连接
2. 填写连接信息：
   - 连接名：自定义标识（如 prod-dm）
   - 主机名：数据库服务器 IP（如 192.0.2.100）
   - 端口号：DM 服务端口（默认 5236）
   - 用户名：SYSDBA（默认超级管理员）
   - 密码：安装时设定的密码
3. 点击"测试"按钮验证连通性
4. 通过后点击"确定"保存
```

> 支持同时管理多个数据库实例，左侧树形导航可自由切换。

#### 2. 表空间管理

```
右键实例 → 管理表空间 → 新建表空间
  ├── 名称：如 TS_APP_DATA
  ├── 数据文件：指定路径和初始大小（如 /dm/data/TS_APP_DATA01.DBF, 初始 128M）
  ├── 自动扩展：开启，每次扩展 64M，最大 32G
  └── 确定创建
```

#### 3. 用户与权限管理

```
右键"用户"节点 → 新建用户
  ├── 用户名：app_user
  ├── 密码：********
  ├── 默认表空间：TS_APP_DATA
  ├── 角色权限页签 → 授予 RESOURCE 角色
  └── 系统权限页签 → 按需授予 CREATE TABLE 等

-- 等效 SQL（可在 SQL 编辑器直接执行）：
CREATE USER app_user IDENTIFIED BY "MyPass123"
  DEFAULT TABLESPACE TS_APP_DATA;
GRANT RESOURCE TO app_user;
```

#### 4. SQL 编辑器

双击实例名展开连接后，右侧主区域即为 SQL 编辑器：

- **快捷键 `Ctrl+Enter`**：执行当前 SQL 或选中 SQL
- **`F8`**：执行并查看执行计划
- **`Ctrl+/`**：注释/取消注释
- **结果集**：支持导出为 CSV / Excel / INSERT 语句
- **历史记录**：已执行的 SQL 自动保存，可回溯

#### 5. 数据库备份与恢复

**前提条件**：执行备份前必须先开启归档模式。

**开启归档模式：**

```sql
-- 在 SQL 编辑器中执行
ALTER DATABASE MOUNT;
ALTER DATABASE ARCHIVELOG;
ALTER DATABASE ADD ARCHIVELOG 'DEST=/dm/arch,TYPE=LOCAL,FILE_SIZE=128,SPACE_LIMIT=10240';
ALTER DATABASE OPEN;
```

**图形化备份步骤：**

```
1. 右键目标实例 → 备份
2. 弹出备份向导，选择备份级别：
   ├── 数据库完全备份
   ├── 表空间备份
   ├── 表备份
   └── 归档日志备份
3. 点击"新建备份"
4. 配置备份参数：
   - 备份集路径：/dm/backup/
   - 备份集名称：DB_FULL_20240116
   - 压缩级别：1（0=不压缩，9=最大压缩）
   - 并行度：2（CPU 核数的一半为宜）
5. 点击"确定"开始备份
6. 在"备份历史"标签页可查看进度和结果
```

**恢复操作：**

```
1. 关闭数据库服务 ./DmServiceXXX stop
2. 打开 DM 控制台工具 ./console
3. 选择"恢复数据库" → 指定备份集路径
4. 执行恢复 → 重启数据库服务
```

#### 6. 代理作业（定时任务）

代理作业功能可以替代操作系统 crontab，实现定时执行数据库任务（备份、统计、清理等）：

**完整设置步骤：**

```
步骤一：创建代理环境
  右键"代理" → 创建代理环境
  系统自动创建 SYSJOB 模式和相关系统表

步骤二：新建作业
  右键"作业" → 新建作业
  ├── 常规页签：
  │   ├── 作业名：DAILY_BACKUP
  │   └── 是否启用：是
  ├── 作业步骤 → 添加：
  │   ├── 步骤名：FULL_BACKUP
  │   ├── 步骤类型：SQL脚本
  │   └── 脚本内容：
  │       BACKUP DATABASE FULL BACKUPSET '/dm/backup/daily_full';
  └── 作业调度 → 添加：
      ├── 调度类型：周期性
      ├── 执行周期：每天
      ├── 执行时间：02:00:00
      └── 确定

步骤三：验证
  右键作业名 → 立即执行 → 查看作业日志确认
```

**常见调度示例：**

```
每天凌晨2点全量备份：
  周期=每天，时间=02:00

每小时清理临时数据：
  周期=每天，间隔=1小时

每周日统计报表：
  周期=每周，星期=周日，时间=06:00
```

#### 7. DMDBG 调试器（PL/SQL 调试）

DM 管理工具内置 PL/SQL 调试器（**DMDBG**），图标为🐢小乌龟，支持存储过程/函数/触发器的图形化调试：

**使用步骤：**

```
1. 在 SQL 编辑器中编写存储过程
2. 右键存储过程名 → "调试"
   ── 或点击工具栏小乌龟图标
3. 调试界面要素：
   ├── 断点：行号左侧单击设置（红色圆点）
   ├── F5：单步跳入（进入嵌套调用）
   ├── F6：单步跳过（不进入嵌套）
   ├── F7：单步返回（跳出当前函数）
   ├── F8：继续执行到下一断点
   └── 变量窗口：实时查看局部变量和参数值
4. 调用堆栈窗口：查看嵌套调用层级
```

**调试示例：**

```sql
-- 创建测试存储过程
CREATE OR REPLACE PROCEDURE calc_bonus(
  emp_id    IN  INT,
  base_pay  IN  DECIMAL(10,2),
  bonus OUT DECIMAL(10,2)
) AS
BEGIN
  IF base_pay > 10000 THEN
    bonus := base_pay * 0.2;  -- 在此行设置断点
  ELSE
    bonus := base_pay * 0.1;
  END IF;
END;

-- 调试调用（在调试器中填入输入参数）
-- {call calc_bonus(1001, 15000, ?)}
```

## DM 服务查看器 (dmservice)

**DM 服务查看器** 是管理达梦数据库服务生命周期的专用工具。

### 启动方式

| 平台 | 启动方法 |
|------|---------|
| **Windows** | 开始菜单 → 达梦数据库 → DM 服务查看器 |
| **Linux** | `cd $DM_HOME/tool && ./dmservice.sh` |
| **命令行** | 需要 `root` 权限启动 |

### 功能操作

#### 服务状态管理

```
选中服务 → 右键菜单：
  ├── 启动服务    —— 启动已停止的数据库服务
  ├── 停止服务    —— 停止正在运行的数据库服务
  ├── 重启服务    —— 先停止再启动
  └── 刷新        —— 更新服务状态显示
```

服务状态图标含义：
- 🟢 绿色：服务正在运行
- 🔴 红色：服务已停止
- 🟡 黄色：服务状态异常

#### 注册新服务

除数据库实例服务外，可注册集群相关服务：

```
右键空白区域 → 注册服务
  ├── 服务名：DmWatchMonitor
  ├── 服务类型：守护进程监控
  ├── 执行路径：$DM_HOME/bin/dmwatcher
  └── 启动参数：ini_path=/dm/data/DAMENG/dmwatcher.ini
```

#### 服务参数修改

```
双击服务名 → 弹出属性窗口
  ├── 常规页签：启动类型（手动/自动/禁用）
  ├── 登录页签：服务运行账户
  └── DmService 参数页签：INI 文件路径、启动参数
```

> **注意**：修改服务参数后需要重启服务才能生效。

## DM 数据迁移工具 DTS (dts)

**DTS（Data Transfer Service）** 是达梦数据库的专业异构数据迁移工具，支持多种主流数据库向 DM 的数据迁移。

### 启动方式

| 平台 | 启动方法 |
|------|---------|
| **Windows** | 开始菜单 → 达梦数据库 → DM 数据迁移工具 |
| **Linux** | `cd $DM_HOME/tool && ./dts` |

### 支持的数据源

**源端（从）：**

| 数据库类型 | 连接方式 | 备注 |
|-----------|---------|------|
| Oracle | JDBC / ODBC | 需安装 Oracle 客户端或配置 JDBC 驱动 |
| MySQL | JDBC / ODBC | 推荐 JDBC 方式 |
| SQL Server | JDBC / ODBC | 注意排序规则（Collation）差异 |
| PostgreSQL | JDBC / ODBC | 注意 schema 概念映射 |
| 达梦 DM | JDBC / ODBC | 同构迁移或版本升级 |
| 文本文件 | 本地文件 | 支持 CSV/TXT/Excel 等格式 |

**目标端（到）：**

| 类型 | 说明 |
|------|------|
| 达梦 DM | 主要目标，完整支持所有 DM 对象类型 |
| 文本文件 | 将数据导出为 CSV/SQL 脚本等格式 |

### 迁移流程

DTS 提供 **5 步向导** 引导式迁移：

```
步骤 1：选择迁移方式
  新建迁移 → 输入项目名称和描述

步骤 2：配置源端连接
  选择数据库类型 → 填写连接信息 → 测试连接

步骤 3：配置目标端连接
  选择达梦 → 填写目标库连接信息 → 测试连接

步骤 4：选择迁移对象
  ├── 模式/用户：选择要迁移的 schema
  ├── 对象筛选：表/视图/存储过程/函数/序列/同义词等
  └── 表数据筛选：可按条件过滤或全量迁移

步骤 5：执行迁移
  查看迁移报告 → 完成
```

### 关键功能详解

#### 迁移评估（预检）

在执行实际迁移前，先进行兼容性评估：

```
DTS 工具栏 → 迁移评估
  ├── 选择源端类型和连接
  ├── 选择评估范围（全库 / 指定 schema / 指定表）
  └── 执行评估

评估报告包含：
  - 对象总数及可迁移率
  - 不兼容对象清单及原因
  - 数据类型映射建议
  - 存储过程/函数兼容性分析
  - SQL 语法差异提示
```

#### 自定义数据类型映射

自动映射不满足需求时，可自定义：

```
默认映射示例：
  Oracle VARCHAR2(100) → DM VARCHAR2(100)

自定义映射需求场景：源字段常有中英文混合，映射到目标需倍增长度

设置步骤：
  工具菜单 → 数据类型映射 → 添加
  ├── 源类型：VARCHAR2
  ├── 源精度：*
  ├── 目标类型：VARCHAR
  └── 目标精度逻辑：2x（表示长度 ×2）
```

#### 分步迁移策略

数据量大的场景（TB 级），推荐分步迁移：

```
第一轮：只迁移表结构
  勾选：表 → 仅勾选"表定义"（不勾选数据/主键/索引/约束）

第二轮：批量迁移数据
  勾选：表 → 仅勾选"数据"
  设置：并行度（4-8个线程）、批量提交行数（1000-5000行/批次）

第三轮：重建索引和约束
  勾选：表 → 仅勾选"主键"+"约束"+"索引"
  原因：数据迁移后再建索引速度更快，避免逐行维护索引开销
```

#### 迁移选项建议

| 选项 | 建议 | 原因 |
|------|------|------|
| 保持对象名大小写 | ✅ 勾选 | 避免 Oracle 习惯的大写对象名被转为小写 |
| 使用默认数据类型映射关系 | ✅ 勾选 | 自动处理常见类型映射，减少手动配置 |
| 迁移前清空目标表 | 视情况 | 重复迁移时使用，首次或增量不宜勾选 |
| 并行度 | 4-8 | 过高可能压垮目标库，建议不超过 CPU 核数 |
| 批量提交行数 | 1000-5000 | 太小效率低，太大可能 OOM |

#### 作业调度

DTS 支持将迁移任务保存为作业并定时执行：

```
迁移项目右键 → 另存为作业
  ├── 作业名：INC_SYNC_DAILY
  ├── 调度：每天 03:00
  └── 保存

此后每天凌晨3点自动执行增量同步。
```

## SQLark 百灵连接

**SQLark（百灵连接）** 是达梦自研的新一代跨平台数据库客户端工具，定位为开发者友好的全能数据库 IDE。

### 获取与安装

| 平台 | 安装方式 |
|------|---------|
| **Windows** | 官网 [www.sqlark.com](https://www.sqlark.com) 下载 `.exe` 安装包 |
| **macOS** | 下载 `.dmg` 安装包，拖入 Applications |
| **Linux** | 下载 `.AppImage` 或 `.deb`/`.rpm` 包安装 |

> SQLark 完全免费，无需 License。

### 核心功能

#### 1. 多数据库连接支持

```
支持的数据库类型：
  ✅ 达梦 DM（原生深度支持）
  ✅ Oracle
  ✅ MySQL / MariaDB
  ✅ PostgreSQL

连接管理：
  - 连接分组（按项目/环境分组管理）
  - SSH 隧道连接（安全访问云上数据库）
  - 连接颜色标记（生产/测试/开发一目了然）
```

#### 2. SQL 智能编辑器

```
编辑器特性：
  ├── 智能代码补全：表名、字段名、函数、关键字自动提示
  ├── SQL 格式化：Ctrl+Shift+F 一键美化
  ├── 外键自动 JOIN：右键表 → "生成外键 JOIN"，自动生成关联查询
  ├── 语法高亮：多数据库方言识别（DM/Oracle/MySQL/PostgreSQL）
  ├── 对象跳转：Ctrl+点击表名直接打开表结构
  ├── 执行计划可视化：树形+图形化展示
  └── 收藏夹：保存常用 SQL 片段
```

**外键自动 JOIN 示例：**

```
-- 有两表：orders (order_id, customer_id) 和 customers (customer_id, name)
-- 右键 orders 表 → "生成外键 JOIN" → 自动生成：

SELECT o.*, c.name AS customer_name
FROM orders o
LEFT JOIN customers c ON o.customer_id = c.customer_id;
```

#### 3. 数据迁移（Oracle → DM）

SQLark 提供比 DTS 更现代的迁移体验：

```
迁移三阶段流程：

阶段一：迁移评估
  - 连接 Oracle 源库
  - 自动扫描所有对象
  - 生成兼容性报告（对象覆盖率、不兼容项清单及修改建议）

阶段二：迁移实施
  - 一键全量迁移
  - 支持断点续迁（网络中断后从断点继续，不重头开始）
  - 一次成功率 90%+
  - 实时显示迁移进度和速率（行/秒）

阶段三：数据校验
  - 自动对比源端和目标端数据
  - 行数校验 + 数据内容抽样校验
  - 输出校验报告，标注差异数据
```

**断点续迁原理：**

```
迁移过程中自动记录检查点：
  已完成表 → 标记为 done
  进行中表 → 记录已迁移行数和主键范围
  未开始表 → 标记为 pending

网络中断恢复后：
  已完成表不重复迁移
  进行中表从上次中断的主键位置继续
  大幅缩短重试时间
```

#### 4. 数据生成

用于生成大规模仿真测试数据：

```
使用场景：
  - 性能测试需要千万/亿级数据
  - 功能验证需要符合业务规则的假数据
  - 演示环境需要逼真的样本数据

操作步骤：
  1. 右键目标表 → "生成数据"
  2. 设置生成行数（如 1000000）
  3. 为每列配置生成规则：
     ├── 姓名列：中文姓名生成器
     ├── 手机号：手机号格式生成器
     ├── 金额列：范围内随机数（100-50000）
     ├── 日期列：日期范围生成器
     └── 外键列：从关联表随机选取
  4. 预览前 100 行确认效果
  5. 执行生成

性能参考：通常速率可达数万行/秒
```

#### 5. 执行计划分析器

```
查看执行计划：
  在 SQL 编辑器中编写查询 → 点击"执行计划"按钮

分析器显示：
  ├── 操作符树：CSCN（全表扫描）、SSEK（索引范围扫描）、HASH JOIN 等
  ├── 每个操作的代价、行数、耗时估算
  ├── 索引使用情况（使用了哪些索引、为何未用其他索引）
  └── 优化建议（建议创建索引的列、建议重写的 SQL 片段）
```

#### 6. ER 图逆向生成

```
从现有数据库反向生成 ER 图：

操作步骤：
  1. 右键数据库/schema
  2. 选择"生成 ER 图"
  3. 选择要包含的表（全选或按需勾选）
  4. 生成可视化 ER 图

ER 图能力：
  - 表间外键关系自动连线
  - 列名、数据类型、主键标注
  - 支持导出 PNG/SVG/PDF
  - 支持缩放、搜索表名定位
```

### manager vs SQLark 对照

| 场景 | 推荐工具 |
|------|---------|
| DBA 日常运维（备份/用户/表空间） | DM 管理工具 (manager) |
| SQL 开发与调试 | SQLark（编辑器体验更现代） |
| Oracle → DM 首次大迁移 | SQLark（断点续迁，成功率更高） |
| 简单同构迁移/导出 | DTS |
| 多数据库环境同时操作 | SQLark（支持 DM/Oracle/MySQL/PG） |
| 服务启停与集群管理 | DM 服务查看器 / dmservice |
