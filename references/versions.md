# 达梦数据库版本区别与获取

## 版本体系

达梦数据库管理系统（DM8）提供三个主要版本：**标准版**、**企业版**、**安全版**。三个版本共享相同的核心数据库引擎，但在功能集、性能上限和安全能力上逐级递增。

### 核心差异对比

| 特性 | 标准版 | 企业版 | 安全版 |
|------|--------|--------|--------|
| **最大连接数** | ≤ 25 | 无限制 | 无限制 |
| **存储容量** | ≤ 500GB | 无限制 | 无限制 |
| **单表行数** | ≤ 1亿行 | 无限制 | 无限制 |
| **DSC 共享存储集群** | ❌ | ✅ | ✅ |
| **MPP 海量并行集群** | ❌ | ✅ | ✅ |
| **RWC 读写分离集群** | ❌ | ✅ | ✅ |
| **DataWatch 数据守护集群** | ❌ | ✅ | ✅ |
| **DPC 分布式集群** | ❌ | ✅ | ✅ |
| **分区表** | ❌ | ✅ | ✅ |
| **列存储（HUGE TABLE）** | ❌ | ✅ | ✅ |
| **闪回查询（Flashback）** | ❌ | ✅ | ✅ |
| **全文索引** | ❌ | ✅ | ✅ |
| **DBLINK（外部链接）** | ❌ | ✅ | ✅ |
| **通讯加密（SSL）** | ❌ | ✅ | ✅ |
| **存储加密** | ❌ | ✅ | ✅ |
| **四权分立** | ❌ | ❌ | ✅ |
| **强制访问控制** | ❌ | ❌ | ✅ |
| **审计与实时侵害检测** | ❌ | ❌ | ✅ |
| **透明/半透明加密** | ❌ | ❌ | ✅ |

### 安全版四权分立详解

安全版在企业版全部功能之上，引入了数据库安全领域的"三权分立"增强版——四权分立，将传统的 DBA 超级权限拆分为四种互不干涉的角色：

| 角色 | 缩写 | 职责 |
|------|------|------|
| **数据库管理员** | DBA | 日常运维管理：创建对象、备份恢复、性能调优，但**无权查看用户数据** |
| **审计管理员** | AUDITOR | 审计配置与日志管理：设置审计规则、查看审计记录、管理审计日志存储 |
| **安全管理员** | SSO | 安全策略制定：用户安全级别标签、强制访问控制策略、数据分类分级 |
| **数据库操作员** | DBO | 数据运维操作：管理数据文件、表空间、日志归档等存储层操作 |

四角色互相监督，任何单一角色都无法完全控制数据库，满足等保三级及以上、涉密信息系统的安全要求。

### 版本选择建议

```
测试/开发环境  →  标准版（功能满足日常开发验证）
生产业务系统  →  企业版（全面功能，集群高可用）
涉密/等保系统  →  安全版（四权分立，强制访问控制）
```

> **注意**：标准版虽不支持集群，但仍可部署单机主备（通过手动日志传输实现冷备，非自动故障切换）。

## 安装包获取

### 官方下载渠道

1. 访问达梦官网生态社区：**[https://eco.dameng.com](https://eco.dameng.com)**
2. 导航至「**下载中心**」→ 选择「**数据库产品**」
3. 根据目标环境选择：
   - **操作系统**：Linux（RedHat/CentOS/麒麟/统信）、Windows、中标麒麟等
   - **CPU 架构**：x86_64、ARM64（aarch64）、龙芯（mips64）、申威（sw64）等
4. 点击下载对应的 ISO 或 ZIP 安装包

### 授权说明

| 用途 | 授权方式 | 获取方式 |
|------|----------|----------|
| **开发/测试** | 免费，无需 License Key | 官网直接下载即可安装使用 |
| **生产部署** | 需要正式 License Key 文件 | 联系达梦销售或代理商购买 |
| **定制需求** | 根据业务定制 | 拨打官方热线 **400-991-6599** |

> 开发版在功能上与正式版完全一致，但授权有效期通常为 1 年，到期后需要重新申请或更换 License Key。

### 安装包命名规则

达梦安装包的命名遵循统一的格式，解析文件名即可了解适用环境：

```
dm8_20240116_x86_rh7_64.iso
 │     │       │    │   │
 │     │       │    │   └── 64位操作系统
 │     │       │    └────── 系统版本缩写（rh7 = RedHat 7 / CentOS 7）
 │     │       └─────────── CPU 架构（x86 / arm / mips / sw）
 │     └─────────────────── 发布日期（YYYYMMDD）
 └───────────────────────── 大版本号（dm8 = 达梦8）
```

**常见系统版本缩写对照：**

| 缩写 | 对应操作系统 |
|------|-------------|
| `rh7` / `rh8` | Red Hat Enterprise Linux 7 / 8、CentOS 7 / 8 |
| `ky10` | 麒麟 V10 |
| `uos20` / `uos1050` | 统信 UOS 20 / 1050 |
| `win64` | Windows 64位 |
| `neokylin7` | 中标麒麟 V7 |

### 安装前环境检查

```bash
# 1. 确认操作系统版本
cat /etc/os-release
# 或
lsb_release -a

# 2. 确认 CPU 架构
uname -m
# 输出: x86_64 / aarch64 / mips64 / sw_64

# 3. 确认系统位数
getconf LONG_BIT
# 输出: 64

# 4. 确认可用磁盘空间（DM 完整安装约需 1-2GB）
df -h /opt

# 5. 确认内存（建议 ≥ 1GB）
free -h
```

### 常见安装错误

| 错误现象 | 原因 | 解决方案 |
|----------|------|----------|
| `无法执行二进制文件` | CPU 架构不匹配（如 ARM 包装到 x86） | 运行 `uname -m` 确认架构，重新下载匹配包 |
| `glibc 版本过低` | 系统 C 库版本不满足要求 | 升级 glibc 或使用匹配的安装包版本 |
| `libXext.so.6: cannot open` | 缺少图形界面依赖（图形化安装时） | `yum install -y libXext libXi libXtst` |
| `Permission denied` | 非 root 用户安装 | 使用 root 用户或 `sudo` 执行安装脚本 |

> **关键原则**：安装包必须与目标操作系统的**发行版、版本号、CPU 架构、系统位数**全部匹配，否则极大概率安装失败或运行不稳定。
